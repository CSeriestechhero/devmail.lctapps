Alrighty, I have a important thing to discuss...
I am planning to move ALL LCTapps projects (not the 1994 STUDIOS ones, since they are right now just Google Slides demos)
There are currently two pages that are hosted via GitHub Pages:
TAH Beta (the app hub beta)
LCTapps Marketplace
We need to figure out if we are going to move all the LCTapps projects over to GitHub, or if some are staying in JSitor. Let me explain the pros (benefits) and cons

BENEFITS FOR JSitor:
Not much, easy hosting, separate HTML, CSS, and JavaScript tabs.

CONS FOR JSitor:
If an error occurs in the code (or sometimes randomly üò≥) The site will not show web page contents AT ALL, which makes the website unusable. This cannot be resolved automatically, you have to usually change something up and pray that it'll work (almost never does). This explains why LCTapps Marketplace could not be hosted via JSitor, and I checked for errors, there was none. The "View source" button at the bottom-right is SOOOOO ANNOYING when you check the webpage.

BENEFITS FOR GitHub Pages:
No random errors will occur except for when the project is being published (that gets resolved automatically üòÄ)
You can upload files in the Git repo (with or without branches) and is easily locate where the file is if needed to be used/called for on a LCTapps project. Lemme give an example:
Lets say we needed a image somewhere for a project. Instead of uploading it via Lettuce.it, or LSC4M2F2, I could add it via the repo and just say the "branch-name/file.png" instead of uploading it via my network, finding the file URL (it takes forever), and then copy-pasting or typing out "https://network.name/eoFERe5fRgrte/file.png"

CONS FOR GitHub Pages:
Errors might occur when updating a page, but that can be fixed easily by a team member who can code.
It's a pain in the butt to host (though I have been practicing and it's getting easier üòè)
GitHub is a bit complicated to learn, but they have a documentation so its all good :) and I've been learning how to GitHub.
Thats probably it!

Overall... I think it's be a good idea to use GitHub because it won't give random, non-fixable errors and I like the environment. Also, the URL's are pretty easy to learn for users when the page is published!
Now, I'd like to hear your opinions on this. Which is better to use?
